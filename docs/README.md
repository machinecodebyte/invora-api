# Backend Documentation

This folder documents how to run, test, maintain, inspect, and extend the
Invora backend.

- `project-runner.md`: step-by-step local setup and run flow
- `commands.md`: daily development commands
- `testing.md`: test strategy and test commands
- `architecture.md`: modular monolith structure and layer rules
- `configuration.md`: local, Docker, and hosted-service runtime configuration
- `database.md`: migrations and VS Code database-client connections
- `deployment.md`: production environment and Docker operations
- `swagger.md`: Swagger UI, ReDoc, OpenAPI JSON, and auth verification notes
- `openapi.md`: configured OpenAPI endpoint reference
- `api-smoke-test.md`: disposable-environment end-to-end API verification flow
- `security-audit.md`: verified controls and remaining security work
- `production-readiness.md`: local verification and staging prerequisites
- `progress.md`: current implementation status and pending modules
